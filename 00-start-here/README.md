# 00 - Start here

## Why this workshop exists

Many teams know GitHub Copilot as “the autocomplete tool.” That is too narrow for modern usage. This workshop shows Copilot as a **developer workflow assistant** across editor chat, reusable context, specs, and the terminal.

## Audience

- technical attendees
- mixed experience with LLMs and AI-assisted coding
- curious about practical adoption, not hype

## What this workshop covers

- what GitHub Copilot is
- how to set it up
- how to use it on simple coding tasks
- how to get better answers through context and instructions
- how advanced features fit into a real development workflow

## What this workshop intentionally skips

- Git basics
- pull requests and code review workflows
- CI/CD
- deployment, except for the optional Azure deployment extension

## How to use the repository

Move through the numbered sections in order. Each section is designed like a mini slide deck:

1. **main message**
2. **short explanation**
3. **demo idea**
4. **try it**
5. **takeaway**

Use the local samples when you want attendees to do real work instead of just watching:

- [Node sample](../samples/node-feedback-api/README.md)
- [Python sample](../samples/python-ticket-analyzer/README.md)
- [Hospital performance sample](../samples/hospital-performance-summary-v2/specs/daily-summary/README.md)

## Success by the end

Attendees should leave with three things:

1. a clear mental model for what Copilot can help with,
2. a practical pair-programming loop for prompting, steering, verifying, and refining,
3. a sense of how to scale usage through shared context and specs.
