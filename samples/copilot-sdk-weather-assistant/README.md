# Copilot SDK weather assistant

Small Python sample for [14 - GitHub Copilot SDK](../../14-github-copilot-sdk/README.md).

It demonstrates:

- starting a Copilot SDK client
- creating a streaming session
- defining a typed local tool
- letting Copilot call that tool during a conversation

## Prerequisites

- GitHub Copilot CLI authenticated locally
- Python 3.11+
- `uv`

Check the CLI first:

```bash
copilot --version
```

## Run

```bash
uv sync
uv run python weather_assistant.py
```

If the Python SDK package is not available for your operating system yet, use this sample as a code walkthrough instead of a hands-on run.

Try:

```text
What's the weather in Seattle and Tokyo?
```

Then:

```text
Which city looks better for an outdoor lunch?
```

Type `exit` to quit.

During a successful run, the terminal prints lines like:

```text
[tool:get_weather] returning demo weather for Seattle
```

That line comes from local Python code. The assistant response is generated by Copilot using the tool result.

## Teaching notes

The weather data is intentionally deterministic. The point of the sample is the SDK flow, not weather accuracy or external API integration.

`PermissionHandler.approve_all` is used only to keep this local workshop sample short. In a real app, permissions should be explicit, user-visible, and scoped to the tool being invoked.
